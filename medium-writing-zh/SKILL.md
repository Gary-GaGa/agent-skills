# Medium 繁體中文寫作與經營指南

## 適用情境

當使用者需要：
- 撰寫或優化 Medium 繁體中文文章
- 規劃 Medium 平台經營策略
- 改善文章排版與閱讀體驗
- 設定 SEO、標籤與曝光策略
- 建立個人品牌與讀者社群

---

## 一、文章結構與寫作框架

### 1.1 標題（Title）

- **字數**：控制在 15～25 字，簡潔有力
- **技巧**：使用數字、疑問句、「如何」句型，或明確承諾價值
- **範例**：
  - ✅「5 個讓 Medium 文章爆紅的排版秘訣」
  - ✅「為什麼你的 Medium 文章沒人看？3 個關鍵原因」
  - ❌「關於寫作的一些想法」（太模糊）
- Medium 標題過長會被截斷，影響推薦卡片顯示

### 1.2 副標題（Subtitle）

- 用一句話補充標題未說完的價值主張
- 可加入關鍵字，強化 SEO
- 字數建議 20～40 字

### 1.3 開頭（Hook）

- 前 2～3 句決定讀者是否繼續閱讀
- 有效手法：
  - 提出痛點或共鳴問題
  - 分享驚人數據或事實
  - 說一個簡短的個人故事
  - 直接點出「讀完你會學到什麼」
- 避免冗長的前言或自我介紹

### 1.4 正文結構

- **一段一重點**：每段只講一個概念
- 善用「總—分—總」或「問題—分析—解法」架構
- 在關鍵轉折處使用過渡句，引導閱讀節奏
- 長文建議加入目錄（使用標題層級即可形成導覽）

### 1.5 結尾（CTA）

- 總結文章核心觀點（1～2 句）
- 加入行動呼籲（Call to Action）：
  - 邀請拍手（Clap）
  - 鼓勵留言分享想法
  - 引導追蹤作者或訂閱
  - 推薦相關文章

## 二、排版與格式規範（好閱讀的關鍵）

### 2.1 段落

- 每段 **2～4 句**（約 3～5 行），避免大段文字牆
- 段落間保留空行，讓版面有呼吸感
- 中文排版不需要首行縮排（Medium 不支援）

### 2.2 標題層級

- **H1**：文章標題（僅一個）
- **H2**：主要章節標題
- **H3**：子章節標題
- 不要跳級（例如 H2 直接跳到 H4）
- 標題要具描述性，避免模糊用詞如「前言」「結語」

### 2.3 列表

- 步驟流程 → 使用**有序列表**（數字）
- 並列重點 → 使用**無序列表**（項目符號）
- 每個列表項目簡潔，1～2 句為佳

### 2.4 強調與標記

- **粗體**：用於核心關鍵字或重點句（每段最多 1～2 處）
- *斜體*：用於引用詞彙、專有名詞或補充說明
- `行內程式碼`：用於技術術語或指令
- 引用區塊（Blockquote）：用於金句、重要觀點或他人言論
- 避免過度使用強調，否則失去效果

### 2.5 分隔線

- 用 Medium 內建分隔線切分大段落或主題轉換
- 適合用在：章節之間、觀點轉換、故事與分析之間

### 2.6 圖片與視覺

- **首圖**：必放，高品質且與主題相關，影響推薦卡片呈現
- 圖片加上說明文字（caption），註明來源
- 建議圖文穿插，每 300～500 字搭配一張圖
- 避免連續多張圖片堆疊
- 圖片 Alt 文字使用中文描述，有助 SEO

### 2.7 中文排版細節

- 中英文之間加半形空格（提升閱讀舒適度）
  - ✅「使用 Medium 撰寫文章」
  - ❌「使用Medium撰寫文章」
- 中文與數字之間加半形空格
  - ✅「共有 5 個步驟」
  - ❌「共有5個步驟」
- 標點符號使用全形中文標點
- 避免一句話過長，超過 40 字建議斷句

## 三、SEO 優化策略

### 3.1 標題 SEO（Meta Title）

- 融入核心關鍵字，放在標題前半段
- 字數 15～25 字
- 確保 Google 搜尋結果不被截斷

### 3.2 描述（Meta Description）

- Medium 的 subtitle 即為 meta description
- 60～100 字，清楚描述文章價值
- 自然融入 1～2 個關鍵字

### 3.3 自訂網址（Custom URL）

- 發布前修改 URL slug 為簡短英文關鍵字
- 例如：`medium.com/@username/medium-writing-tips`
- 避免使用 Medium 自動生成的亂碼 URL

### 3.4 內文關鍵字佈局

- 核心關鍵字出現在：標題、副標題、第一段、H2 標題
- 使用同義詞和相關詞彙自然分佈
- 避免關鍵字堆砌，保持語意自然

### 3.5 圖片 SEO

- 所有圖片加上中文 Alt 文字
- 描述圖片內容並融入關鍵字

## 四、標籤（Tags）策略

### 4.1 基本規則

- Medium 每篇文章最多 **5 個標籤**
- 標籤組合建議：
  - **熱門標籤** ×1～2：有大量追蹤者的標籤（如 `Writing`、`Self Improvement`）
  - **利基標籤** ×1～2：精準觸及目標讀者（如 `Medium教學`、`繁體中文寫作`）
  - **地區標籤** ×1：鎖定本地市場（如 `台灣`、`華文`）

### 4.2 繁體中文常用標籤參考

| 類型 | 標籤範例 |
|------|----------|
| 寫作 | `寫作`、`Writing`、`繁體中文寫作`、`自媒體` |
| 行銷 | `內容行銷`、`Content Marketing`、`SEO` |
| 個人成長 | `自我成長`、`生產力`、`閱讀心得` |
| 技術 | `程式設計`、`Software Development`、`AI` |
| 經營 | `Medium教學`、`部落格經營`、`個人品牌` |

### 4.3 標籤優化技巧

- 觀察同主題熱門文章使用的標籤
- 優先選擇已有追蹤者數量的標籤
- 中英文標籤混用，以中文為主
- 定期檢視標籤成效，根據數據調整

## 五、平台經營策略

### 5.1 發文頻率

- 建議每週 1～2 篇穩定產出
- 品質優先於數量
- 固定發文時間有助於養成讀者習慣

### 5.2 出版物（Publications）

- 投稿至相關主題的出版物，借力既有讀者群
- 繁體中文熱門出版物值得關注與加入
- 可自建出版物經營品牌，彙整系列文章

### 5.3 讀者互動

- 回覆每一則留言，建立關係
- 主動閱讀、拍手、留言其他作者的文章
- 善用 Highlights 功能增進互動

### 5.4 跨平台導流

- 將文章分享至 Facebook 社團、LINE 群組、PTT、Threads
- 使用 Friend Link 讓非付費用戶也能閱讀付費牆文章
- 經營電子報（Newsletter），將讀者留在自己的名單

### 5.5 數據分析

- 定期查看 Medium Stats：
  - **Views**：文章曝光數
  - **Reads**：完整閱讀數
  - **Read Ratio**：完讀率（目標 ≥ 30%，優秀 ≥ 50%）
  - **Fans**：拍手讀者數
- 完讀率低 → 檢查文章長度、開頭吸引力、排版
- 曝光高但完讀低 → 標題吸引但內容未達預期

## 六、Medium Partner Program 收益

### 6.1 申請條件

- 擁有 Medium 帳號並加入會員
- 至少發布一篇文章
- 連結 Stripe 收款帳戶

### 6.2 收益計算方式

- 主要依據：**付費會員在你文章上的閱讀時間**
- 互動（拍手、留言、Highlights）也會影響分潤
- 投稿至出版物可增加曝光與閱讀時間

### 6.3 提升收益技巧

- 寫深度長文（1,500～3,000 字），增加停留時間
- 提高完讀率比追求瀏覽量更重要
- 善用系列文章引導讀者閱讀更多內容

## 七、文章模板

### 7.1 教學文模板

```
# [數字] 個 [主題] 的 [結果/技巧]

[1～2 句 Hook：痛點或引人好奇的問題]

[1 句話說明讀者將獲得什麼]

---

## 1. [第一個重點]

[說明 + 範例或圖片]

## 2. [第二個重點]

[說明 + 範例或圖片]

## 3. [第三個重點]

[說明 + 範例或圖片]

---

## 結語

[1～2 句總結核心價值]

[CTA：邀請讀者互動]
```

### 7.2 經驗分享文模板

```
# [個人經驗描述] — [獲得的啟發或成果]

[用故事或場景開頭，讓讀者有畫面]

---

## 背景

[描述情境、挑戰或問題]

## 我做了什麼

[具體步驟或決策過程]

## 學到的事

[可量化的成果 + 反思]

---

## 給你的建議

[可執行的行動建議]

[CTA]
```

### 7.3 觀點評論文模板

```
# [針對某趨勢/現象的觀點聲明]

[Hook：引述新聞、數據或現象]

---

## 現況是什麼

[客觀描述現象]

## 我的觀察

[個人觀點 + 論證]

## 這代表什麼

[延伸影響或預測]

---

## 你可以怎麼做

[讀者可採取的行動]

[CTA]
```

## 八、寫作檢查清單

發布前逐項確認：

- [ ] 標題是否簡潔有力、包含關鍵字？
- [ ] 副標題是否補充了標題的價值？
- [ ] 開頭前 3 句是否能抓住注意力？
- [ ] 每段是否只講一個重點、不超過 5 行？
- [ ] 是否使用了 H2/H3 標題層級？
- [ ] 是否有列表或引用來打破文字牆？
- [ ] 粗體/斜體是否適度使用？
- [ ] 中英文之間是否有空格？
- [ ] 首圖是否高品質且相關？
- [ ] 圖片是否有說明文字和 Alt 文字？
- [ ] URL slug 是否已自訂為簡短英文？
- [ ] 是否選擇了 5 個合適的標籤？
- [ ] 結尾是否有明確的 CTA？
- [ ] 在手機上預覽排版是否舒適？

## 九、常見錯誤與避免

| 錯誤 | 正確做法 |
|------|----------|
| 段落過長（超過 7 行） | 拆分為 2～4 句一段 |
| 沒有小標題 | 每 200～400 字加一個 H2 或 H3 |
| 標題模糊無吸引力 | 具體、有數字或動詞的標題 |
| 只用英文標籤 | 以中文標籤為主，混搭英文 |
| 沒有首圖 | 必放高品質首圖 |
| 忽略 SEO 設定 | 自訂 URL、subtitle 融入關鍵字 |
| 發文後不互動 | 主動回覆留言、與社群互動 |
| 追求量忽略質 | 每篇文章都應有明確價值 |
